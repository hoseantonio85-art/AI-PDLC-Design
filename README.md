# AI PDLC Design

Короткая продуктовая система для работы с LLM в дизайне и проектировании интерфейсов.

Проект помогает продукт-оунерам, дизайнерам и AI-инструментам не прыгать сразу в UI, а сначала договориться о смысле задачи: зачем она нужна, какую проблему решает, кто пользователь, какой сценарий должен измениться и как это связано с продуктом.

## Зачем это нужно

Часто задача приходит в формате «сделать экран» или «собрать прототип». Из-за этого LLM и инструменты вроде Lovable начинают генерировать интерфейс раньше, чем понятны цель, проблема и пользовательский сценарий.

Этот набор файлов помогает:

- быстрее вводить LLM в контекст продукта;
- валидировать постановку задачи до дизайна;
- не придумывать новые сервисы без необходимости;
- учитывать философию продукта, карту сервисов и UX-паттерны;
- готовить более точные промпты для генерации прототипов.

## Как устроен проект

```text
product-system/
├── 00_start_here/          # входной мастер-промпт
├── 01_product_foundation/  # философия, миссия, цель сервиса
├── 02_services/            # карта сервисов и описания модулей
├── 03_ux/                  # UX-принципы и паттерны
├── 04_ui/                  # базовые UI-правила
├── 05_examples/            # хорошие и плохие примеры задач
└── 06_lovable/             # контекст и шаблон handoff для Lovable
```
