# Lovable context

Мы проектируем систему управления рисками как интеллектуального партнёра пользователя.

Интерфейс должен:

- показывать ситуацию, а не просто данные;
- объяснять, почему это важно;
- вести пользователя к действию;
- использовать прогрессивное раскрытие;
- сначала показывать summary, потом детали.

Основные паттерны:

- список для обзора и приоритизации;
- дровер для просмотра без потери контекста;
- модалка для сфокусированного действия;
- AI-блок для объяснений и рекомендаций.

Не создавай новые сервисы без прямого указания. Не превращай интерфейс в отчётную таблицу. Не перегружай первый экран.
